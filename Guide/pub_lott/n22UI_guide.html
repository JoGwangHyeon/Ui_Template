#레이어팝업
팝업

슬라이드팝업과, 풀팝업, alert, confirm등 as-is팝업의 마크업과 기능이 그대로
유지되었고, 스타일만 수정되었습니다.

그외 추가로 제작된 별도의 기능이 있는 팝업에 대한 가이드 입니다.

기간선택 팝업

(안드로이드 기준)네이티브로 제작된 기간선택 피커와 디자인을 맞춰놓은상태이며,
간편설정과 직접입력 메뉴가 둘다 존재하는 팝업인지,
둘중하나만 존재하는 팝업인지 여부에 따라 3가지로 구분되며,
관련 스크립트는 common_n22UI.js파일내에 hanaProdUI.dialSelect 함수 입니다.
해당 함수는 개발에서 한번더 커스텀되어 적용되어있습니다.

개발에서 전달되는 값은 아래와 같습니다.

#3D 모션이미지
모션 영역의 종류

상품몰 고도화 프로젝트에서
모션 영역이 3가지로 구분됩니다.

1. 이미지를 조각내서 css로 모션을 적용한 케이스
2. lottie용 json을 업체에서 전달받아 적용한 케이스
3. 1번과 2번을 섞어놓은 케이스
모션 json 설정관련

json파일의 경우 to-be 공통 함수에 정지 프레임이 설정되어있으며,
추가로 전달된 파일에 정지 프레임을 설정하고자 할경우 아래와 같이 호출하면 됩니다.

setSVGApply('#prd_detail_04_4203_01', '/pub/resource/images/C/lottie/detail/J49.json', false, false, false, 33);

false로 선언된 부분들은 인수인계 문서에 가이드 되어있으며,
33이라고 적혀진 부분이 정지 프레임값을 임의로 설정하는 곳입니다.

모션 json 크기 관련

json 파일들이 규격화 되어 제작 되지않아,
같은 너비와 높이를 적용하여 화면에 노출시켜도 오브젝트의 크기가 다릅니다.
해당 부분은 추후 개선되어야할 부분이며,
오브젝트 변경시 주변 오브젝트와 사이즈를 맞춰주는 작업이 필요합니다.

이벤트

모션영역들은 스크롤 이벤트 발생시
화면 노출여부에 따라 자동재생 또는 정지 되도록
화면별로 스크립트가 정의되어있습니다.
(상품상세는 prd_detail_n22UI.js 파일내 hanaProdDetailUI.prdDetailPageInit 함수에 정의) 화면에 노출되는지 체크하여 있다면, 타겟 요소에 motion-init클래스가 적용되며 모션을 시작합니다.

#레이아웃

대부분 as-is와 동일한 레이아웃을 사용하고 있으며,
특별히 가이드가 필요한 부분은 대부분 없으나,
메인, 상품몰 서브메인, 가입프로세스 정보입력 화면은 가이드가 필요할듯하여
기록합니다.

메인, 서브메인

화면 진입시 hanaProdUI.sliderAutoplay 함수가
#content 요소에 1초뒤 motion-init 함수를 적용하고,
그후 2초뒤 hero-motion-finish 클래스를 적용합니다.

motion-init 클래스가 적용되면 css로 모션을 진행하며,
스크롤 시 요소마다 is-load 클래스가 부여되면 등장하는
모션의 speed나 easing은 원큐 메인과 싱크를 맞춘 상태입니다.

가입프로세스 - 정보입력

입출금전환을 제외하고 예금, 적금, 청약 모두
가입 프로세스의 단계별 화면이 swiper slide item으로 구성되어있습니다.
사용자가 다음 화면으로 swipe 하기 위해서는 [계속] 버튼을 클릭하여야하며,
마지막 스탭에서는 [계속] 버튼이 [다음] 버튼으로 변경됩니다.

개발에서 사용자가 입력한 데이터의 검증 처리후 화면이동이나 스크롤 이동등을 위해,
개발에서 사용될 함수가 작성되어있으며,
각 정보입력화면의 하단 소스와 주석을 참조하시면 됩니다.

참고사항
공통 css

화면마다 공통으로 인클루드 되고있는
resource/css/common.css
resource/css/components.css
resource/css/bank.css
위 3개 파일은 원큐앱 공통이므로 수정하면 안됩니다.

css상의 safe-area-inset-top 과 safe-area-inset-bottom

원큐 iOS 앱으로 확인가능한
safe-area-inset-top 과 safe-area-inset-bottom 영역으로인해
스크립트에서도 해당 영역을 구하여
스크롤시에 추가로 더하거나 빼며 연산하고 있으며,

css상에서도
top:calc(3.5rem + env(safe-area-inset-top))
와 같이 선언하여 더하거나 빼서 안드로이드와
화면을 일치시키고 있습니다.

G5기종에서는 해당 css 연산문구를 해석하지 못하여 속성 선언자체를 무시하므로,
top:3.5rem;
top:calc(3.5rem + env(safe-area-inset-top));
와 같이 두번 선언해주어야합니다.
